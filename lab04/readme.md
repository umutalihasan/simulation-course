# Лабораторная работа №4 — Базовый датчик случайных чисел

Эта лабораторная про одну простую, но интересную идею: можно ли с помощью двух арифметических операций получить числа, которые ведут себя как случайные? Спойлер — да, и довольно хорошо.

---

## Что нужно было сделать

Задача была такая: реализовать свой генератор псевдослучайных чисел, сравнить его со встроенным в Python, и проверить — насколько оба близки к теоретическим значениям равномерного распределения U(0,1).

Звучит просто, но за этим стоит целый пласт математики.

---

## Генератор — формула из лекции

Мы использовали **мультипликативный конгруэнтный генератор (MCG)**, он же генератор Лемера. Формула была дана прямо на слайде:

```
x*_i = (β · x*_{i-1}) mod M
x_i  = x*_i / M
```

Первая строчка — это сам генератор, который гоняет числа по кругу через умножение и взятие остатка. Вторая — нормализация, чтобы получить вещественное число от 0 до 1.

### Параметры (именно те, что в лекции)

| Параметр | Значение |
|----------|----------|
| M | 2⁶³ = 9 223 372 036 854 775 808 |
| β | 2³² + 3 = 4 294 967 299 |
| x*₀ | β = 4 294 967 299 |
| Период | M / 4 = 2⁶¹ |

Важно: M здесь **не простое число**, это степень двойки. Из-за этого правила выбора параметров немного другие, чем в классическом варианте.

---

## Почему именно эти параметры

Когда M = 2^k, для максимального периода нужно выполнить два условия:

1. **β ≡ 3 или 5 (mod 8)** — проверяем: (2³² + 3) mod 8 = 3 ✔
2. **Начальное значение x*₀ должно быть нечётным** — β = 4 294 967 299 нечётное ✔

Если оба условия выполнены — получаем максимальный период M/4 = 2⁶¹. Это примерно 2.3 × 10¹⁸ уникальных чисел до повторения. Для нашей выборки в 100 000 значений — более чем достаточно, цикла не будет вообще.

Почему именно mod 8? Потому что при M = 2^k структура мультипликативной группы повторяется с периодичностью, кратной 8. Это результат из теории чисел: только β с остатком 3 или 5 при делении на 8 порождает подгруппу максимального размера M/4.

---

## Что происходит внутри генератора

На каждом шаге происходит буквально два действия:

```python
self.x = (self.beta * self.x) % self.M   # умножение + остаток
return self.x / self.M                    # нормализация в (0, 1)
```

Результат — число в интервале (0, 1). Почему именно равномерное распределение? Потому что x* при правильных параметрах обходит все значения от 1 до M-1 ровно по одному разу за полный период. Когда делим на M — все они ложатся в (0,1) равномерно, без пропусков и сгущений.

---

## С чем сравнивали

Второй генератор — встроенный `random.random()` в Python. Под капотом там **вихрь Мерсенна (MT19937)** с периодом 2¹⁹⁹³⁷−1. Это один из лучших общедоступных PRNG.

Оба генератора производят числа из U(0,1), поэтому сравнение честное. Именно это важно — хоса специально предупреждал: нельзя сравнивать генераторы, если они производят разные распределения. Нормализация через деление на M как раз и обеспечивает эту честность.

---

## Теоретические значения

Для U(0,1) они вычисляются аналитически:

**Среднее:**
```
μ = (a + b) / 2 = (0 + 1) / 2 = 0.5
```

**Дисперсия:**
```
σ² = (b - a)² / 12 = 1 / 12 ≈ 0.08333
```

Дисперсия 1/12 — это не магия, это результат интегрирования. Если считать честно:

```
σ² = ∫₀¹ (x - 0.5)² dx
```

Делаем замену t = x - 0.5, получаем ∫ t² dt от -0.5 до +0.5, это t³/3 на границах:

```
(0.5³ - (-0.5)³) / 3 = (0.125 + 0.125) / 3 = 0.25 / 3 = 1/12
```

12 не берётся из воздуха — оно вылезает из геометрии самого распределения.

## Что посчитали

Выборка: **100 000 значений** (минимум по условию задачи).

Для каждого генератора посчитали выборочное среднее и дисперсию:

```
x̄ = (1/n) · Σ xᵢ
s² = (1/n) · Σ (xᵢ - x̄)²
```

Итого получилось **6 чисел — 3 пары**, как и требовал преподаватель:

| | Среднее | Дисперсия |
|---|---|---|
| Теория U(0,1) | 0.500000 | 0.083333 |
| MCG (наш) | ~0.500001 | ~0.083337 |
| Python built-in | ~0.499381 | ~0.083281 |

Пары сравниваются попарно: теория vs MCG, теория vs built-in, MCG vs built-in.

## Серия экспериментов

Кроме основного эксперимента, прогнали генераторы на разных объёмах выборки — от 100 до 1 000 000 значений. Смысл в том, чтобы наглядно показать **Закон больших чисел**: чем больше N, тем ближе выборочные характеристики к теоретическим.

| N | Погрешность среднего (MCG) |
|---|---|
| 100 | ~0.016 |
| 1 000 | ~0.008 |
| 10 000 | ~0.002 |
| 100 000 | ~0.0003 |
| 1 000 000 | ~0.0001 |

Погрешность убывает пропорционально 1/√N — это и есть закон больших чисел в действии.

## Свойства генератора

### Математические
- Производит **равномерное распределение U(0,1)**
- Теоретическое среднее: 0.5, дисперсия: 1/12
- Период: 2⁶¹ — повторений в нашей выборке нет
- При росте N значения сходятся к теоретическим

### Программные
- **Детерминированный**: одинаковый seed → одинаковая последовательность
- **Не потокобезопасен**: два потока на одном объекте сломают состояние
- **Не криптографический**: зная несколько выходных значений, можно восстановить seed
- **Seed ≠ 0 и нечётный**: иначе либо весь ряд нули, либо период резко падает

### Алгоритмические сложности

| Операция | Время | Память |
|---|---|---|
| Инициализация | O(1) | O(1) |
| Одно число | O(1) | O(1) |
| N чисел | O(n) | O(n) |
| Среднее / дисперсия | O(n) | O(1) |

O(1) на каждое число — это принципиально важно для симуляций, где нужны миллиарды значений.

## Вывод

MCG — алгоритмически простой генератор: одно умножение и один остаток от деления. Но при правильно подобранных параметрах он даёт результаты, практически неотличимые от встроенного генератора Python на выборке в 100 000 значений. Оба сходятся к теоретическим значениям U(0,1), что подтверждает закон больших чисел.

MCG не подходит для криптографии и имеет более короткий период, чем Mersenne Twister. Но для задач моделирования и симуляции — это быстрое, надёжное и предсказуемое решение.
