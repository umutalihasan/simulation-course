## Назначение работы

Цель проекта - продемонстрировать моделирование случайных событий двух типов:

1. **Бинарное событие** - приложение **«Скажи “Да” или “Нет”»**.
2. **Событие из группы несовместных исходов** - приложение **«Шар предсказаний» (Magic 8-Ball)**.

В обоих случаях программа не использует `Math.random()` в основной логике принятия решений. Вместо этого применяется собственный **мультипликативный конгруэнтный генератор** случайных чисел, что делает результаты воспроизводимыми при фиксированном `seed`.

## Теоретическая основа

### 1. Базовый датчик случайного события

Пусть `α` - случайное число из интервала `[0,1)`, а `p` - вероятность наступления события `A`. Тогда:

```text
P{α < p} = p
```

Следовательно, если сгенерировать `α` и проверить условие `α < p`, можно смоделировать событие с вероятностью `p`.

### 2. Генерация события из полной группы

Если даны несовместные события `A1, A2, ..., Am` с вероятностями `p1, p2, ..., pm`, где:

```text
p1 + p2 + ... + pm = 1,
```

то выбор события можно выполнить методом последовательного вычитания вероятностей:

```text
A := α
A := A - p1
если A <= 0, произошло событие A1
иначе A := A - p2
если A <= 0, произошло событие A2
...
```

### 3. Статистическая обработка

После серии из `N` испытаний вычисляется:

- `nk` - число появлений события `Ak`;
- `p̂k = nk / N` - эмпирическая вероятность.

Это позволяет сравнить теоретические вероятности с результатами моделирования.

## Описание реализации

## 1. Генератор случайных чисел (`lcg.js`)

В проекте используется **мультипликативный конгруэнтный генератор**:

```text
Xₙ₊₁ = (16807 · Xₙ) mod 2147483647
```

Параметры генератора:

- `mult = 16807`
- `mod = 2147483647`
- `seed ≠ 0`

Особенности реализации:

- внутреннее состояние хранится в `state`;
- при вызове `next()` вычисляется следующее состояние генератора;
- результат нормализуется в интервал `[0,1)`;
- при одном и том же `seed` получается одна и та же последовательность чисел.

Это особенно важно для демонстрации, тестирования и повторяемости результатов.

---

## 2. Часть 1 - «Скажи “Да” или “Нет”»

### Назначение

Пользователь задаёт вероятность ответа **«Да»**. Вероятность ответа **«Нет»** рассчитывается автоматически:

```text
p(Нет) = 1 - p(Да)
```

### Принцип работы

1. Генерируется число `alpha = LCG.next()`.
2. Проверяется условие `alpha < pYes`.
3. Если условие истинно, результат - **ДА**.
4. Иначе результат - **НЕТ**.

### Статистика

Для этой части ведутся:

- `stats1[0]` - число ответов «Да»;
- `stats1[1]` - число ответов «Нет»;
- `N1` - общее число испытаний.

При запуске симуляции на `N` испытаний программа многократно выполняет генерацию и затем вычисляет эмпирические вероятности:

```text
p̂(Да)  = n(Да) / N
p̂(Нет) = n(Нет) / N
```

## 3. Часть 2 - «Шар предсказаний»

### Назначение

Во второй части моделируется более сложное случайное событие. Все ответы разделены на три группы:

- **positive** - положительные ответы;
- **neutral** - нейтральные ответы;
- **negative** - отрицательные ответы.

Пользователь задаёт вероятности двух первых групп:

```text
pPos, pNeu
```

Вероятность третьей группы вычисляется автоматически:

```text
pNeg = 1 - pPos - pNeu
```

### Двухэтапный выбор

Генерация результата выполняется в два шага:

#### Шаг 1. Выбор группы

Генерируется `alpha`, после чего применяется алгоритм последовательного вычитания:

```text
A := alpha
A := A - pPos
если A <= 0 -> positive
иначе A := A - pNeu
если A <= 0 -> neutral
иначе -> negative
```

#### Шаг 2. Выбор ответа внутри группы

После выбора группы выполняется второй вызов генератора:

```js
idx = Math.floor(LCG.next() * list.length)
```

Затем из выбранной группы берётся конкретный текст ответа.

### Статистика

Для второй части хранится массив счётчиков по каждому конкретному ответу. После серии запусков рассчитываются:

- абсолютные частоты каждого ответа;
- относительные частоты каждого ответа;
- сводные частоты по группам `positive / neutral / negative`.

## Алгоритмы работы

### Алгоритм части 1

```text
Ввод: pYes
Сгенерировать alpha ∈ [0,1)
Если alpha < pYes:
    результат = «Да»
Иначе:
    результат = «Нет»
Обновить статистику
```

### Алгоритм части 2

```text
Ввод: pPos, pNeu
Вычислить pNeg = 1 - pPos - pNeu
Сгенерировать alpha ∈ [0,1)
Определить группу методом последовательного вычитания
Сгенерировать второе случайное число
Выбрать конкретный ответ внутри группы
Обновить статистику
```

### Алгоритм симуляции на N испытаний

```text
Обнулить статистику
Повторить N раз:
    сгенерировать случайное событие
    увеличить соответствующий счётчик
Для каждого исхода вычислить Frequency[k] = Statistics[k] / N
```

## Примеры результатов

Ниже приведены реальные результаты, полученные при фиксированном `seed = 12345`.

Это важно, так как фиксированный `seed` позволяет воспроизводить одну и ту же последовательность случайных чисел и демонстрировать стабильность работы алгоритма.

### 1. Часть 1 - «Да / Нет»

**Параметры:**

- `p(Да) = 0.65`
- `p(Нет) = 0.35`
- `N = 1000`
- `seed = 12345`

**Результат симуляции:**

```text
Общее N           = 1000
n(Да)             = 666
n(Нет)            = 334
p̂(Да)            = 0.666
p̂(Нет)           = 0.334
Последнее state   = 1408436056
```

**Интерпретация:**  
Полученная эмпирическая вероятность `0.666` близка к теоретической `0.65`. Небольшое отклонение объясняется конечным объёмом выборки и естественным статистическим разбросом.

### 2. Часть 2 - одиночный запуск «Шара предсказаний»

**Параметры:**

- `pPos = 0.50`
- `pNeu = 0.28`
- `pNeg = 0.22`
- `seed = 12345`

**Первый результат:**

```text
alpha             = 0.0966165285
Выбранная группа  = positive
Ответ             = Знаки говорят - да
Последнее state   = 1790989824
```

**Интерпретация:**  
Поскольку `alpha < 0.50`, была выбрана положительная группа. После этого вторым вызовом генератора выбран конкретный ответ внутри группы `positive`.

### 3. Часть 2 - симуляция на 1000 испытаний

**Параметры:**

- `pPos = 0.50`
- `pNeu = 0.28`
- `pNeg = 0.22`
- `N = 1000`
- `seed = 12345`

**Групповые результаты:**

```text
Общее N           = 1000
positive          = 499   -> p̂(+) = 0.499
neutral           = 276   -> p̂(~) = 0.276
negative          = 225   -> p̂(-) = 0.225
Последнее state   = 1538715103
```

**Интерпретация:**  
Эмпирические вероятности `0.499 / 0.276 / 0.225` близки к заданным значениям `0.50 / 0.28 / 0.22`, что подтверждает корректность моделирования.

## Обоснование корректности

Реализацию можно считать корректной по следующим причинам:

1. **Бинарное событие** моделируется строго по формуле `P{α < p} = p`.
2. **Групповое событие** моделируется методом последовательного вычитания вероятностей, что соответствует теории генерации события из полной группы несовместных исходов.
3. **Статистическая обработка** выполнена через накопление счётчиков и вычисление относительных частот.
4. **Фиксированный seed** обеспечивает повторяемость результатов и удобство тестирования.
5. **Эмпирические частоты** при больших `N` приближаются к теоретическим вероятностям, что подтверждается примерами запуска.
